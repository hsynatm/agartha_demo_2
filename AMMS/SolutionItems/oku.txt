## Log seviye stratejisi (HBYP)

Development -> Graylog:
  Information, Warning, Error, Fatal
  Debug yalnizca VS Output (Console)'da kalir

Production -> Graylog:
  Warning, Error, Fatal
  Information Graylog'a gitmez

HTTP request log seviyeleri:
  2xx / 3xx -> Information
  4xx -> Warning
  5xx / exception -> Error

## Serilog konfig dosyalari

- appsettings.Development.json -> Graylog restrictedToMinimumLevel: Information
- appsettings.Production.json -> Graylog restrictedToMinimumLevel: Warning
- appsettings.json -> Graylog host/port (localhost:12201)

## Graylog arama ornekleri

Tum dev API loglari:
  Application:AMMS.Api AND EnvironmentName:Development

CorrelationId ile izleme:
  CorrelationId:<header-degeri>
  (HTTP request, EF Core, exception ve audit loglarinda AmmsGraylogSchemaEnricher ile ayni deger)

## Graylog log semasi (standart alan seti)

Tum log tipleri (200/400/404/500/exception) AmmsGraylogSchemaEnricher ile ayni kolonlari tasir.
Veri yoksa alan cikarilmaz; string alanlar "-", numerik alanlar 0 kullanilir.

Zorunlu alanlar:
  Application, ClientIp, CorrelationId, Elapsed, ElapsedMilliseconds, EnvironmentName,
  ErrorCode, LocalizationKey, ModuleName, RequestMethod, RequestPath, SourceContext,
  StatusCode, TenantId, Title, Detail, Host, Scheme, TraceId, message, source,
  stringLevel, timestamp

Opsiyonel (her zaman dolu; exception yoksa "-"):
  ConnectionId, ExceptionMessage, ExceptionSource, ExceptionType, StackTrace, RequestId

ErrorCode / LocalizationKey / Title / Detail:
  - 2xx: "-" (AmmsLogSchemaContext.SetFromRequest)
  - 4xx/5xx: ExceptionHandlingMiddleware -> AmmsLogSchemaContext.SetFromMapped

Pipeline sirasi (onemli):
  CorrelationId -> Culture -> LogContext -> RequestLogging -> ExceptionHandling
  ExceptionHandling controller'a yakin; request log final StatusCode ile duser.

Modul bazli:
  ModuleName:FaultManagement

Hata takibi (Warning+):
  Application:AMMS.Api AND (StatusCode:>=400 OR level:>=4)

Validation hatalari:
  Application:AMMS.Api AND ErrorCode:VALIDATION_ERROR

Audit (dev):
  Application:AMMS.Api AND message:"Audit entry written"

## Token alma (Postman / test)

POST http://localhost:8080/realms/amms/protocol/openid-connect/token
Content-Type: application/x-www-form-urlencoded

grant_type=password
client_id=amms-spa
username=admin
password=admin

Yanittaki access_token ile API cagrisi:
  Authorization: Bearer {access_token}

## Beklenen davranis

- Token yok -> 401 Unauthorized
- Gecerli token -> 200 + CreatedBy/UpdatedBy + Graylog UserId/TenantId dolu
- /health -> AllowAnonymous (token gerekmez)
